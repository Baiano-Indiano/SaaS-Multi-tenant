# Plan 41-01: Weekly Email Digests & PDF/JSON Server Reporting

**Goal:** Create a weekly cron email digest endpoint via QStash, implement server-side PDF generation using `jspdf`, and add a report exports interface on the settings activity page.

<threat_model>
- **Threat 1:** Data leakage from reports compilation.
  - *Mitigation:* Explicitly validate session ownership and restrict route access using the standard RBAC `requirePermission` utility before compiling reports.
- **Threat 2:** SSRF or signature spoofing on Cron endpoints.
  - *Mitigation:* Protect `/api/cron/*` endpoints using secure QStash signature verification via `@upstash/qstash` receiver logic.
</threat_model>

## Tasks

### Wave 1: Server-Side PDF & Activity Reports Generation

<task id="41-01-01" name="Install jspdf and Implement Report Generator">
  <read_first>
    - src/lib/db/schema.ts
  </read_first>
  <action>
    Install `jspdf` via `npm install jspdf`.
    Create `src/lib/reports/generator.ts`.
    1. Implement `generateOrgReportData(orgId: string): Promise<any>` using Drizzle query helpers (e.g. counting projects, audit logs, and member changes in the last 7 days).
    2. Implement `generatePdfReport(orgName: string, data: any): Buffer` using `jspdf` to draw a structured layout report containing organization metrics and activity logs. Return the output as a Node.js `Buffer`.
    Write unit tests under `src/lib/reports/__tests__/generator.test.ts` to test compiling PDF report structures without throwing errors.
  </action>
  <acceptance_criteria>
    - File `src/lib/reports/generator.ts` exists.

    - Tests under `src/lib/reports/__tests__/generator.test.ts` pass successfully.
  </acceptance_criteria>
</task>

<task id="41-01-02" name="Create Reports Download Route">
  <read_first>
    - src/lib/auth/rbac-utils.ts
  </read_first>
  <action>
    Create `src/app/api/org/[orgSlug]/reports/route.ts`.
    1. Validate session. Verify that the user has admin access (`org:update` or `org:read`) to the organization slug.
    2. Read `format` query param (`?format=pdf` or `?format=json`).
    3. Generate the data using `generateOrgReportData(orgId)`.
    4. Compile the target format asynchronously. Return the payload with attachments headers:
       - For PDF: `Content-Type: application/pdf`, `Content-Disposition: attachment; filename="audit-report.pdf"`
       - For JSON: `Content-Type: application/json`, `Content-Disposition: attachment; filename="activity-logs.json"`
  </action>
  <acceptance_criteria>
    - Route `/api/org/[orgSlug]/reports` returns the correct download files.
    - Returns 401/403 for unauthenticated or unauthorized requests.
  </acceptance_criteria>
</task>
