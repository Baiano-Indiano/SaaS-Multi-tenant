# Plan 43-01: Automatic Data Retention Policies (GDPR/LGPD)

**Goal:** Implement automated data retention policy settings for audit logs, including database schema updates, UI settings panel integration, server action verification, and a QStash daily background sweep job.

<threat_model>
- **Threat 1: Privilege Escalation on Data Deletion Settings**
  - *Mitigation:* Strictly enforce RBAC check `security:manage` in the data retention update action. Sanitize and validate the inputs using Zod to enforce positive integers >= 7.
- **Threat 2: Unauthenticated Sweep execution (Denial of Service / Bulk Deletion)**
  - *Mitigation:* Secure the `/api/cron/cleanup-logs` route using `@upstash/qstash` signature verification or Bearer token validation matching `CRON_SECRET`.
- **Threat 3: SQL Injection on Dynamic Schema Deletion Query**
  - *Mitigation:* Ensure that the dynamic schema name is retrieved from the authenticated/validated database record `organizations.tenantSchemaName` and matches a strict regex pattern `/^tenant_[a-zA-Z0-9_]+$/` before constructing SQL query templates.
</threat_model>

<schema_push_requirement>

### Wave 3: Back-end Sweep Cron Job

<task id="43-01-07" name="Implement cleanup-logs cron job route">
  <read_first>
    - src/app/api/cron/billing-sync/route.ts
    - src/lib/db/tenant-db.ts
  </read_first>
  <action>
    Create file `src/app/api/cron/cleanup-logs/route.ts`.
    1. Setup QStash receiver verification or CRON_SECRET auth.
    2. Query all organizations where `dataRetentionDays IS NOT NULL AND dataRetentionDays > 0`.
    3. Loop in batches. For each organization:
       - Validate dynamic schema name regex `/^tenant_[a-zA-Z0-9_]+$/`.
       - Connect to organization dynamic schema and execute:
         `DELETE FROM "${tenantSchema}".audit_log WHERE "createdAt" < NOW() - INTERVAL '${retentionDays} days'`
       - Record system audit log of the sweep run execution metadata (e.g. number of rows purged).
  </action>
  <acceptance_criteria>
    - API route `/api/cron/cleanup-logs` exists.
    - Cron requires valid auth headers or QStash signature.
    - Safely executes hard delete matching the configured retention days.
  </acceptance_criteria>
</task>
