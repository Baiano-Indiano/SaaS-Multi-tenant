# Plan 39-01: Backend Connectors & OAuth Routing

**Goal:** Implement encryption utilities, API connectors, and callback API endpoints for Slack and Microsoft Teams OAuth handshakes.

<threat_model>
- **Threat 1:** Integration token leakage from database backups or unauthorized read access.
  - *Mitigation:* Encrypt all credentials using AES-256-GCM before saving to database.
- **Threat 2:** CSRF attacks during OAuth callback redirects.
  - *Mitigation:* Generate signed JWT for the `state` parameter containing `userId` and `orgId`, verified in the callback before completing setup.
</threat_model>

## Tasks

### Wave 1: Core Integrations & Encryption

<task id="39-01-01" name="Implement Token Encryption & State JWT">
  <read_first>
    - src/lib/db/schema.ts
  </read_first>
  <action>
    Create `src/lib/integrations/encryption.ts`.
    1. Implement `encrypt(text: string): string` using native Node.js `crypto` with `aes-256-gcm`. Generate a random 12-byte IV. Derive a 32-byte key from `process.env.CONNECTOR_SECRET` using `crypto.scryptSync`. Return output formatted as `${ivHex}:${authTagHex}:${encryptedHex}`.
    2. Implement `decrypt(encryptedText: string): string` parsing the hex parts and returning decrypted string.
    3. Implement `generateStateToken(userId: string, orgId: string): string` signing a token using `jsonwebtoken` with `expiresIn: '15m'` and key `process.env.CONNECTOR_SECRET`.
    4. Implement `verifyStateToken(token: string): { userId: string, orgId: string }` returning verified payload.
  </action>
  <acceptance_criteria>
    - File `src/lib/integrations/encryption.ts` exists.
    - Exported functions `encrypt`, `decrypt`, `generateStateToken`, `verifyStateToken` exist.
  </acceptance_criteria>
</task>

<task id="39-01-02" name="Implement Slack OAuth Client and Callback API">
  <read_first>
    - src/lib/db/schema.ts
  </read_first>
  <action>
    Create `src/lib/integrations/slack.ts` to manage Slack API calls. Import `WebClient` from `@slack/web-api` and use `getTenantDb` for database operations.
    1. Create `src/app/api/connectors/slack/authorize/route.ts`. Check session. Generate signed state token using `generateStateToken(userId, orgId)`. Redirect user to `https://slack.com/oauth/v2/authorize?client_id=${process.env.SLACK_CLIENT_ID}&scope=chat:write,incoming-webhook&state=${state}`.
    2. Create `src/app/api/connectors/slack/callback/route.ts`. Retrieve `code` and `state`. Verify state using `verifyStateToken(state)`. In exchange for `code`, request access token from `https://slack.com/api/oauth.v2.access` using `WebClient`. Save access token encrypted (via `encrypt`) in `connector` table of the tenant schema with `type: 'slack'`.
  </action>
  <acceptance_criteria>
    - File `src/lib/integrations/slack.ts` exists.
    - Route `src/app/api/connectors/slack/authorize/route.ts` exists.
    - Route `src/app/api/connectors/slack/callback/route.ts` exists.
  </acceptance_criteria>
</task>

<task id="39-01-03" name="Implement Teams OAuth Client and Callback API">
